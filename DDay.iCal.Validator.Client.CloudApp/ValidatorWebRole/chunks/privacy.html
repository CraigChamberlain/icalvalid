What the Validator saves

When you use the URL method of validation, the Validator assigns an ID to each calendar you validate, it saves the calendar along with the validation results, and it displays a permalink to the saved calendar and results.

Note that the permalink that represents the results of an URL-based validation is semi-private. You can can use the permalink, or share it, but it's unprotected and could be discovered by someone else.

What the Validator does not save

  1. Calendars submitted using File Upload or Snippet.
  2. The IP addresses or user-agent strings associated with users of URL-based validation.

Rationale for saving calendars and results

If you use URL-based validation, the data you are sending to the validator is already public at the URL you specify. Results are saved in this case in order to help improve the iCalendar ecosystem. Specifically:

  • A public namespace of correct and incorrect iCalendar examples will facilitate discourse about the examples.
  • Analysis of iCalendar examples created by a broad range of iCalendar-producing components will be helpful to the authors of those components.

How to opt out

If you want to validate without permitting data collection, use one of the transient methods: File Upload or Snippet.
